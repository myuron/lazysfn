# lazysfn 仕様書

AWS Step Functionsの情報をターミナルから確認できるTUIツール。lazygit風のUI/UXでStep Functionsの稼働状況を閲覧する。

## 概要

- **言語**: Go
- **TUIライブラリ**: jroimartin/gocui
- **操作範囲**: 閲覧のみ（実行の開始・停止などの操作機能は不要）
- **参考プロジェクト**: lazygit

## 画面遷移

### 1. 起動 → Profile選択

- アプリ起動時、profile選択モーダルのみを画面中央に表示する（メイン画面はまだ描画しない）
- モーダルサイズ: 幅40文字、高さはprofile数+2（枠線分）、最大で画面高さの80%
- `~/.aws/config` からprofile一覧を読み込み、profile名のみをリスト表示する
- `j/k` でカーソル移動、`Enter` で選択
- `q` でアプリ終了

### 2. Profile選択 → ステートマシン一覧ロード

- profile選択後、モーダルを閉じてメイン画面を描画する
- メイン画面にスピナーを表示し、ステートマシン一覧のロードを開始する
- ロード完了後、左パネルにステートマシン一覧を表示する
- ステートマシンが0件の場合、左パネルに `(0 state machines)` と表示する
- ロード中に `Esc` を押すとキャンセルし、profile選択モーダルに戻る

### 3. エラー時

- エラーメッセージをモーダル（画面中央、幅60文字、高さはメッセージ行数+2）で表示する
- AWS SDKが返すエラーメッセージをそのまま表示する
- AWS接続エラー、`~/.aws/config` のパースエラー、スロットリングエラー等、すべてのエラーを同じモーダルで扱う
- MVP時点ではスロットリングエラーもエラーモーダルで表示する（後続リリースでエクスポネンシャルバックオフに置き換え）
- `Enter` または `q` でprofile選択モーダルに戻る

### 4. ステートマシン選択 → 実行履歴表示

- 左パネルでカーソルを移動すると、右パネルに選択中のステートマシンの実行履歴をロード・表示する
- MVP時点ではカーソル移動のたびにAPIを呼び出す。後続リリースでデバウンス（300ms）を導入して最適化する
- ロード中は右パネルにスピナーを表示する

### 5. リフレッシュ設定

- 専用キーでリフレッシュ設定ポップアップを開閉する

## AWS認証

- 起動時に `~/.aws/config` のprofile一覧をリスト表示し、ユーザーが選択する
- リージョンは選択したprofileに紐づくものを使用

## APIレートリミット対策

### MVP

- 実行履歴は直近20件を初期ロードし、必要に応じて追加ロードする（一括取得しない）
- APIエラー（スロットリング等）が発生した場合、エラーモーダルで表示しprofile選択に戻す

### 後続リリース

- 左パネルのカーソル移動時、300msのデバウンスを適用し、連続移動中はAPIコールを抑制する
- 自動リフレッシュのデフォルト間隔は5秒とし、AWS APIのレートリミットに抵触しない範囲とする
- APIエラー（スロットリング等）が発生した場合、エクスポネンシャルバックオフでリトライする
- リトライ中はユーザーに不快感を与えないよう、UIをブロックせずバックグラウンドでリトライし、ステータスバー等で状態を通知する

## パネル構成

パネルの横幅比率は左:右 = 1:3（例：ターミナル幅120の場合、左30文字・右90文字）。

### 左パネル: ステートマシン一覧

- ステートマシン名を一覧表示
- 各ステートマシンの右側に直近の実行結果ステータスを色付き記号 `●` で表示（色はステータスの色分けに準拠）
- ステートマシン名が長い場合は末尾を切り詰めて `...` を表示し、`●` の表示領域を確保する
- ステートマシン名は `ListStateMachines` 完了後に先に表示し、直近ステータスは取得でき次第順次表示する
- 表示順序: 名前でソート
- インクリメンタルサーチで絞り込み可能 [後続リリース]

### 右パネル: 実行履歴

左パネルで選択したステートマシンの実行履歴を表示する。一番上が最新の実行。

表示カラム（左から右の順）:

| カラム | 幅 | 備考 |
|---|---|---|
| 実行ID | 30文字固定 | 長い場合は切り詰めて `...` を表示 |
| 実行結果 | 10文字 | 色付き |
| 失敗ステート名 | 20文字 | FAILED / TIMED_OUT / ABORTED 時のみ表示、それ以外は空欄 |
| 開始時間 | 19文字 | `2006-01-02 15:04:05` 形式 |
| 終了時間 | 19文字 | `2006-01-02 15:04:05` 形式。RUNNING状態は `-` を表示 |
| 動作時間 | 10文字 | `1h23m45s` 形式。RUNNING状態は開始時間から現在までの経過時間を表示 |
| Input Param | 残りの幅 | 長い場合は切り詰めて `...` を表示 |

- 初期表示は直近20件
- リスト末尾で下へのキー操作が連続して検知されたら追加ロード [後続リリース]

## ステータスの色分け

| ステータス | 色 |
|---|---|
| SUCCEEDED | 緑 |
| FAILED | 赤 |
| RUNNING | 青 |
| TIMED_OUT | 黄 |
| ABORTED | グレー |

## キーバインド

Vim準拠のキーバインド体系。

### グローバル（全画面共通）

| キー | 動作 | MVP |
|---|---|---|
| `q` | アプリ終了（ポップアップ表示中はポップアップを閉じる） | o |
| `?` | キーバインドヘルプをポップアップで表示/閉じる | - （後続リリース、MVPでは無視） |
| `s` | リフレッシュ設定ポップアップを開く/閉じる | - （後続リリース、MVPでは無視） |
| `R` | 手動リフレッシュ（ステートマシン一覧 + 選択中の実行履歴の両方を再取得） | o |

### メイン画面

| キー | 動作 | MVP |
|---|---|---|
| `j` / `k` | カーソルを下/上に移動 | o |
| `h` / `l` | 左パネル/右パネルにフォーカス移動 | o |
| `Tab` | フォーカスを次のパネルに切り替え | o |
| `/` | インクリメンタルサーチモードに入る（左パネルのみ） | - （後続リリース、MVPでは無視） |

### 検索モード

| キー | 動作 |
|---|---|
| 文字入力 | 検索文字列を更新し、リアルタイムで絞り込み |
| `Esc` | 検索を解除し、全件表示に戻る |
| `Enter` | 検索結果を確定し、通常モードに戻る（絞り込み状態を維持） |

### Profile選択モーダル

| キー | 動作 |
|---|---|
| `j` / `k` | カーソルを下/上に移動 |
| `Enter` | 選択したprofileで接続 |
| `q` | アプリ終了 |

### リフレッシュ設定ポップアップ

| キー | 動作 |
|---|---|
| `j` / `k` | 項目間を移動（ON/OFF切り替え、間隔入力） |
| `Enter` | ON/OFFトグル / 間隔入力欄にフォーカス |
| 数値入力 | リフレッシュ間隔を秒単位で入力 |
| `Esc` / `s` / `q` | ポップアップを閉じる |

### キーバインドヘルプポップアップ

| キー | 動作 |
|---|---|
| `?` / `Esc` / `q` | ポップアップを閉じる |

## リフレッシュ機能

- 専用のポップアップパネルでON/OFF切り替え
- リフレッシュ間隔はユーザーが任意で設定可能（数値入力、秒単位）
- デフォルト間隔: 5秒

## AWS API連携

AWS SDK: `aws-sdk-go-v2` を使用する。対象はStandardタイプのステートマシンのみ（Expressは対象外）。

### 使用するAPI

| 用途 | API | 備考 |
|---|---|---|
| Profile一覧取得 | `~/.aws/config` を直接パース | `[profile xxx]` セクションを抽出してprofile名一覧を取得。`[default]` も選択肢に含める |
| ステートマシン一覧 | `sfn:ListStateMachines` | 全件取得（ページネーション対応）。クライアント側で `type == STANDARD` のもののみフィルタ |
| 直近の実行結果 | `sfn:ListExecutions` | ステートマシンごとに `maxResults=1` で最新1件のステータスを取得 |
| 実行履歴 | `sfn:ListExecutions` | 選択中のステートマシンに対し `maxResults=20` で取得。追加ロード時はページネーショントークンを使用 |
| 失敗ステートの特定 | `sfn:GetExecutionHistory` | FAILED / TIMED_OUT / ABORTED の実行に対して `reverseOrder=true` で呼び出す。`TaskFailed`, `ExecutionFailed`, `LambdaFunctionFailed`, `ActivityFailed`, `MapRunFailed` のいずれかに該当するイベントを探し、該当イベントの `previousEventId` をたどって `stateEnteredEventDetails.name` からステート名を取得する。取得できない場合は空欄とする |
| 実行の入力パラメータ | `sfn:DescribeExecution` | 実行のInput Paramを取得 |

### データ取得フロー

1. **起動時**: `~/.aws/config` をパースしてprofile一覧を表示
2. **profile選択後**: `ListStateMachines` で全ステートマシンを取得（ページネーション対応）
3. **ステートマシン一覧表示時**: 各ステートマシンに対し `ListExecutions(maxResults=1)` で直近の実行ステータスを取得
4. **ステートマシン選択時**: `ListExecutions(maxResults=20)` で実行履歴を取得
5. **実行履歴の各行**: FAILED / TIMED_OUT / ABORTED の実行に対して `GetExecutionHistory` でエラー発生ステートを特定
6. **実行履歴の各行**: `DescribeExecution` でInput Paramを取得
7. **追加ロード時**: ページネーショントークンを使い次の20件を取得

### 注意事項

- 手順3では、ステートマシンの数だけAPIコールが発生する。ステートマシン数が多い環境では並行度を制限する（例: 最大10並列）
- 手順5, 6も実行履歴の件数分APIコールが発生する。同様に並行度を制限する
- 自動リフレッシュ時は手順2〜6を再実行する

## リリーススコープ

### 初回リリース（MVP）

- Profile選択モーダル（`~/.aws/config` からprofile一覧表示、選択）
- ステートマシン一覧（左パネル、名前ソート）
- 直近の実行ステータス表示（色付き）
- 実行履歴表示（右パネル、直近20件、カラム表示）
- 手動リフレッシュ（`R` キー）
- エラーハンドリング（AWS接続エラー時のモーダル表示、profile選択への復帰）

### 後続リリース

- インクリメンタルサーチ（`/` キー）
- デバウンス付きカーソル連動（300ms）
- 自動リフレッシュ + 設定ポップアップ（`s` キー）
- キーバインドヘルプ（`?` キー）
- 実行履歴の追加ロード（末尾でのキー操作連続検知）
- エクスポネンシャルバックオフによるリトライ
